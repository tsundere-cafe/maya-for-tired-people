Houdini Terrain Workflow

Saturday

sketched a test area in photoshop

made some really quick geometry in probuilder (focussed on getting the scale and level of detail representative)

merged and exported as .obj for loading into houdini as test data

wrote the actual houdini code (this part took about 5h i think and i didn't really document it)

you can build a digital asset for unity by selecting everything, hitting "collapse selected into subnetwork", then stepping out and hitting "create digital asset" and saving that into your unity Assets folder somewhere.

you can edit the properties that are exposed to unity by right clicking again and hitting "type properties"

drag whatever properties you want to expose to unity from the Create Parameters box into the Existing Paremeters box, and save by right-clicking the node and hitting "Save Node Type"

back in unity you can drag the new digital asset from your assets folder out into the hierarchy, then drag your probuilder meshes into "input nodes", configure the asset parameters however you want and hit Bake GameObject, it should comes out basically the same in unity as it did in houdini, if you're processing terrain you need "Keep World Transform" set or it'll zero everything out before processing
